1 Introducción

En esta asignatura se utilizará el lenguaje de programación Python para la mayoría de las prácticas debido a la simplicidad de su sintaxis y a la gran variedad de bibliotecas que vienen incorporadas al lenguaje y que hacen posible el abordar los problemas planteados en estas prácticas desde diferentes niveles de abstracción.

En esta primera sesión se indica cómo instalar un entorno de desarrollo basado en Python, para que el alumno que lo desee pueda hacerlo también en su ordenador particular, y se introduce someramente su sintaxis y filosofía.

Ten en cuenta que esta asignatura usa Python3 en lugar de Python2, que se comporta de forma diferente en lo que respecta a la entrada/salida de cadenas, entre otras cosas1.

2 Instalación

No necesitas instalarlo en la máquina de prácticas, pues ya viene. Esta sección explica cómo instalarlo en otras plataformas.

Dependiendo del sistema operativo en que se quiera trabajar, podemos encontrar:

  • En OSX Python ya viene preinstalado, pero la versión que trae es la 2.7. No obstante, ya que en esta asignatura usaremos Python3, que no es sencillo de instalar en OSX, junto con otras herramientas y servidores propios del mundo linux, que tampoco serían sencillos de instalar en OSX, la mejor opción para un alumno que use este operativo es que utilice VirtualBox (versión para OSX) para crear e instalar una máquina virtual con Ubuntu Server, como se describe en otro guión de prácticas.
  • En la mayoría de las distribuciones de Linux también viene preinstalado, o bien se instala con un simple sudo apt-get install python. En Ubuntu 16.04 LTS (tanto en la versión escritorio como la de servidor) se instala también python 3.5 de forma automática. Cuando tecleas python a secas la versión que se intenta cargar es la 2.7, que no viene preinstalada por lo que dará un error. Para usar la versión 3, que es la que necesitamos, hay que teclear python3.
  • En Windows la alternativa típica suele ser descargar un instalador específico para Windows desde la página de descarga de python. Sin embargo este enfoque requiere tener permisos de administrador para posteriormente poder ejecutar dicho instalador. Por ello, otra alternativa popular es la instalación de una versión “portable”. Este tipo de versiones no requieren una instalación propiamente dicha, sino simplemente que sean copiadas a una carpeta cualquiera desde la cual se ejecutarán (funcionan también desde un lápiz usb), y ya que no modifican archivos de sistema no requieren permisos de administración.

El siguiente apartado explica cómo instalar una versión portable para windows, por si quieres hacerlo en tu casa para llevar siempre contigo (en un lápiz USB) un entorno python que pueda ser usado en cualquier ordenador Windows. No obstante, no necesitas hacer esto en los ordenadores de prácticas en los que ya está instalado. Además en esta asignatura haremos casi todas las prácticas en Linux, donde también viene preinstalado como ya se ha dicho.

2.1 Instalación de python portable en Windows

Lo que se describe en esta sección no es necesario para esta asignatura, pues Python ya está instalado en los ordenadores de prácticas. Es sólo por si quieres hacerlo en tu ordenador personal.

  1. Ve a la página de descarga y descarga la versión más reciente de WinPython 3.4.x.x.
  2. Obtendrás un fichero ejecutable (.exe) que es en realidad un archivo comprimido auto-extraíble. Al ejecutarlo te preguntará dónde quieres extraerlo en tu disco duro. Aunque podrías darle como destino un lápiz USB no se recomienda hacerlo en este punto pues la escritura en las memorias flash es muy lenta. Dale como destino una carpeta de tu disco duro, preferiblemente en el raíz de la primera unidad de disco (C:\).
  3. Una vez se ha extraído al destino que has especificado, puedes copiar la carpeta resultante a cualquier otro disco, incluyendo un lápiz USB externo.

Con esta copia la instalación está completada. De ahora en adelante, a la carpeta en la que se ha copiado WinPython la llamaremos _WinPython_ en este documento. En tu caso el nombre real de la carpeta puede ser por ejemplo C:\WinPython 3.4.4.3\, de modo que cuando más adelante en el documento nos refiramos a la carpeta _WinPython_\App, se trataría en tu caso de la carpeta C:\WinPython 3.4.4.3\App.

2.2 Probando Python

Python es un lenguaje interpretado. Esto quiere decir que va leyendo una a una las órdenes que tú le des (bien sea directamente en una ventana del intérprete o bien en un fichero de texto en el que has escrito previamente el programa), y las va ejecutando. Si alguna de las líneas no puede ejecutarse debido a un error, la ejecución se abortará en ese punto.

Por tanto hay dos formas de probar python:

  1. Abriendo el intérprete sin especificarle ningún archivo a ejecutar, y tecleando directamente en el intérprete las órdenes que queremos que ejecute. Este método es rápido para hacer pequeños experimentos, pero tiene el inconveniente de que lo que hayamos escrito se pierde al abandonar el intérprete.
  2. Escribiendo previamente en un archivo las órdenes que componen nuestro programa y después ejecutando el intérprete pasándole como argumento el nombre del fichero que hemos escrito. Es costumbre poner a estos ficheros extensión .py.

Aún hay una tercera que consiste en escribir el programa en un fichero .py y pasárselo como argumento al intérprete, pero con una opción que hace que cuando termine de ejecutar ese programa, en lugar de finalizar, el intérprete se quede a la espera de nuevas órdenes, que recibiría en este caso directamente desde el teclado. Esta tercera opción es como una mezcla de las dos anteriores, ya que nos permite escribir en un fichero definiciones de funciones y variables con las que después podremos experimentar interactivamente en la ventana del intérprete.

Vamos a probar ambas formas como sigue:

2.2.1 Probar el intérprete interactivo

El intérprete se llama python3. En sistemas Unix basta abrir una terminal y teclear python3 para entrar en ese intérprete y poder comenzar a darle comandos. En sistemas Windows, si python ha sido instalado desde su instalador oficial, éste habrá modificado el PATH del sistema de modo que al abrir una interfaz de comandos de Windows (cmd) también será posible teclear python3 directamente desde esa ventana.

En cambio, ya que hemos instalado la versión portable de python y ésta no modifica nada en el sistema, el PATH no estará actualizado con la ruta hacia la carpeta donde haya sido instalado, y tendremos que especificar dicha ruta “a mano” al lanzar el intérprete desde una terminal. Haz lo siguiente:

  1. Abre una interfaz de comandos (Ejecutar → cmd)

  2. En ella escribe: _WinPython_\App\python.exe. Obtendrás un resultado como el siguiente:

     

  3. Prueba a escribir cualquier expresión matemática entre números (el operador de exponenciación es **) y verás el resultado. En modo intérprete interactivo, cada vez que python evalúa una expresión muestra en pantalla el resultado, sin que sea necesario hacer ningún print()

2.2.2 Escribiendo el programa

Abre un editor de texto cualquiera y escribe un fichero llamado prueba.py con el siguiente contenido:

nombre = input("Como te llamas: ")
print("Hola", nombre)

La función input() lee del teclado la respuesta del usuario, hasta que este pulse retorno de carro, y devuelve la cadena tecleada (incluyendo el retorno de carro final)2. Si se le pasa un argumento a esta función, lo imprimirá a modo de prompt antes de leer la respuesta. En el código anterior asignamos la cadena respuesta a la variable nombre para seguidamente mostrarla por pantalla con un print()3. Cuando ejecutas un programa python desde un fichero, a diferencia del intérprete interactivo, no se muestran automáticamente los resultados tras cada línea, sino que es necesario hacer un print() explícito cuando queramos ver algún resultado.

En la ventana de la interfaz de comandos usa cd (cambiar de directorio) hasta situarte en la carpeta en la que hayas guardado prueba.py, y ejecuta el siguiente comando:

> _WinPython_\App\python.exe prueba.py

Deberías ver cómo python te pide el nombre para saludarte a continuación. Estas pruebas sirven para comprobar que el intérprete está instalado correctamente.

2.3 Instalación de un IDE

Aunque, como ves, todo lo que necesitas para escribir y probar programas en python es un editor de textos y una interfaz de comandos, resulta por lo general más cómodo el poder utilizar un editor específico para el lenguaje que te ofrezca coloreado sintáctico, autocompletado de identificadores, etc, y si además tiene integrado un mecanismo que permita ejecutar el intérprete python y mostrar los resultados en un panel del editor, ya no sería necesario ni abrir una interfaz de comandos para probar el código.

Existen multitud de IDEs específicos para python, y otros IDEs de propósito general famosos como Eclipse también pueden ser usados con python. Incluso Visual Studio tiene un plugin que permite trabajar con Python, creando un entorno muy amigable con auto-completado, documentación on-line, etc.

Cuando trabajes con python en la máquina virtual Ubuntu, no podrás usar ninguno de estos IDEs, puesto que la versión Server que usamos no tiene display gráfico. En este caso el “ide” será un editor de texto cualquiera (por ejemplo vim) en el que escribirás el código, y la línea de comandos desde la que lo ejecutarás.

3 Introducción al lenguaje Python

Python es un lenguaje que puede usarse a muchos niveles diferentes. Internamente es un lenguaje orientado a objetos, pero es posible usarlo sin esa característica, sino como un mero lenguaje de scripting como hemos hecho en el ejemplo anterior, o en un nivel ligeramente superior, como un lenguaje procedural tipo C, pero débilmente tipado. En python las variables no tienen tipo, son los datos los que pueden ser de diferentes tipos, pero las variables no son más que referencias a los datos y una misma variable puede usarse para referenciar tanto un entero como una cadena como cualquier otro tipo.

Por otro lado tiene incorporado de forma nativa el tipo de datos list que hace muy sencillo trabajar con listas (o arrays) que crecen dinámicamente a medida que se necesita, y los operadores de manejo de listas hacen que se pueda usar en ocasiones como un lenguaje funcional. Las funciones son un tipo de dato más, y por tanto pueden pasarse como parámetro o declararse dentro de otras funciones, para implementar closures (si bien de forma no enteramente satisfactoria).

Pero la gran ventaja de python es la gran cantidad de bibliotecas (módulos usando su terminología) que vienen ya instalados con el lenguaje y a las que se accede con una sintaxis uniforme. Python es un lenguaje muy adecuado para escribir prototipos rápidos que ayudan a comprender un problema y su posible solución, y probar estas ideas antes de pasar a implementarlas en un lenguaje más convencional como C, C++, C# o Java. Es especialmente adecuado para escribir clientes o servidores de red, ya que proporciona no sólo acceso a la API de sockets “pura”, sino también varios envoltorios de más alto nivel para acceder a protocolos de aplicación frecuentemente utilizados, tales como HTTP, SMTP y otros que veremos en esta asignatura.

En esta sección no se puede cubrir todos los aspectos de python, que son inabarcables en una asignatura, sino sólo la sintaxis básica para trabajar con variables, bucles, condiciones, funciones y clases, con especial hincapié en el manejo de las listas y diccionarios que son casi el tipo de datos “básico” usado por el resto de sus bibliotecas.

A lo largo de futuras sesiones de prácticas se irán cubriendo más aspectos del lenguaje a medida que sean necesarios.

3.1 Datos, variables y expresiones

3.1.1 Tipos

Cada dato en python tiene un tipo concreto. Los tipos más importantes que usaremos son los siguientes. Usa la función type() sobre cada uno de ellos en el intérprete, para ver cómo se llaman en python cada uno de ellos.

  • Entero: 5
  • Entero grande: 7123917237128
  • Flotante: 5.0
  • Cadena: "Hola", o bien 'Hola' (da igual). Puede contener cualquier carácter Unicode (hay que especificar la codificación usada por nuestro editor en un comentario inicial que diga # coding: utf8, por ejemplo)
  • Secuencia de bytes: b"Hola" similar a cadena, pero no intenta “entender” utf8 ni otros encodings, simplemente es la secuencia de bytes que el editor haya guardado.
  • Lista: [1,2,3]
  • Tupla: (1,2,3)
  • Diccionario (también llamado array asociativo): { "nombre": "Manolo", "apellido": "Salas" }

Una observación sobre el tipo entero. Por defecto python usa 32 bits para almacenar un int, lo que en otros lenguajes de programación como el C se denomina un entero largo (long int). Pero a diferencia de otros lenguajes, si el resultado de una operación no cabe en esos 32 bits, en lugar de producirse un error por desbordamiento, python cambia automáticamente el tipo del resultado a long, que es un tipo especial de entero que no tiene restricciones de tamaño y en el que caben números tan grandes como te permita la memoria del sistema. Como curiosidad intenta ver el resultado de 2**1000, donde ** es el operador de exponenciación. De modo que no confundas el tipo long de python con el long de otros lenguajes.

Las variables en cambio no tienen tipo. No son más que una referencia a un dato, y por tanto el tipo de una variable es el tipo del dato al que apuntan. No se declaran de antemano, pero sí deben estar inicializadas antes de usarse por primera vez. El ámbito de las variables es local a la función en que aparecen por primera vez, o al módulo si se inicializan fuera de las funciones.

>>> x
Traceback (most recent call last):
  File "<interactive input>", line 1, in <module>
NameError: name 'x' is not defined
>>> x = 3
>>> x
3
>>> type(x)
<type 'int'>
>>> x = "Hola"
>>> type(x)
<type 'str'>
>>> x
'hola'
>>> print(x)
hola

En Python3 los nombres de variable (o funciones, o clases; en realidad cualquier identificador) pueden estar formados por caracteres Unicode. Es posible por tanto tener una variable llamada año, sin tener que usar rodeos como anyo. No obstante recuerda que si tu código fuente contiene algún carácter no ascii, ya sea en los identificadores, o en las cadenas entre comillas o incluso en los comentarios, debes especificar en un comentario en la primera línea qué codificación usa tu editor, para que Python sepa cómo convertir esas secuencias de bytes a Unicode válido. Por ejemplo # coding: utf8.

3.1.2 Expresiones

Python evalúa las expresiones como cualquier otro lenguaje orientado a operadores (ejemplos: C, Java). Algunas particularidades que se pueden probar en el intérprete:

  • Exponenciación: 2**8
  • Suma de strings: "Hola" + "mundo"
  • Producto de string por entero (repite concatenar): "Hola"*3
  • División: 5/2 (el resultado será 2.5)4
  • División entera: 5//2 (el resultado será 2)
  • Módulo: 5%2
  • Los enteros son tan grandes como se necesite: 2**1000
  • Interpolación de cadenas (tipo printf): "Peso: %.2f Kg, Altura: %d cm" % (68.5, 170). En general: cadena % tupla. Si la tupla tiene un solo elemento pueden omitirse los paréntesis

3.1.3 Acceso a los elementos de una lista, cadena o tupla

Se puede acceder a un elemento de una lista, o cadena, o tupla usando corchetes, pero también dentro de los corchetes se puede usar una expresión tipo [inicio:fin] para obtener una copia de una serie de elementos contiguos (el “fin” nunca se incluye). Ejemplo:

>>> t = "hola"
>>> t[0]
'h'
>>> t[0:3]
'hol'
>>> t[1:3]
'ol'

Si se omite inicio se asume cero. Si se omite fin se asume la longitud de la cadena, lista o tupla. Si un índice es negativo, python le suma la longitud de la cadena, lista o tupla. Así [-1] representa el último elemento, [-2] el penúltimo, etc.

3.1.4 Mutabilidad

Cuando asignamos un nuevo valor a una variable, cambiamos a dónde apunta, no el dato que “contiene”. La analogía variables como cajas no es válida en python. Es mejor la analogía variables como etiquetas que se le pegan a un dato. x=2 pega una etiqueta x en el dato 2. Más adelante hacer x = 5, despega la etiqueta x del dato 2 y se la pega al dato 5.

Los datos en si son inmutables. No puedo cambiar un 2 por un 5. El 2 siempre será 2, y el 5 siempre será 5. Puedo pegarles diferentes etiquetas. La asignación x = y simplemente pega la etiqueta x al mismo dato que tenía ya la etiqueta y. Ese dato tiene ahora dos etiquetas (referencias, alias, …)

Esto se hace importante en las listas y los diccionarios, ya que estos tipos SON MUTABLES. Una lista es una colección de datos (cada uno puede ser de un tipo diferente), a los que se accede por un índice (como los arrays). En realidad es una lista de referencias a datos. Cada elemento de una lista puedo hacerlo apuntar a un dato distinto del que apuntaba inicialmente. Por eso la lista es mutable. Ejemplo:

>>> l = [1,2,3]
>>> l
[1, 2, 3]
>>> l[0]
1
>>> l[0] = 9
>>> l
[9, 2, 3]

l es una referencia que apunta a una lista. l[0] es otra referencia que apunta al dato 1, pero después la cambio para que apunte al dato 9. Como consecuencia la lista ha cambiado. Por eso es mutable. Observa además lo siguiente:

>>> x = l
>>> x[0] = 2
>>> l
[2, 2, 3]

La asignación x = l hace que x apunte a la misma lista que l. Son dos etiquetas para la misma lista. Por tanto modificar x[0] afecta a l. Esto es importante cuando se pasen listas como parámetros de funciones. Si se quiere evitar esto, hay que sacar una copia de la lista, en lugar de asignarla. Esto se logra de dos formas:

>>> x = [1,2,3]
>>> x
[1, 2, 3]
>>> y = list(x)
>>> y[0]=0
>>> y
[0, 2, 3]
>>> x
[1, 2, 3]

O bien, mediante un “slice” parcial o total de la lista:

>>> y = x[:]
>>> y[0]=5
>>> y
[5, 2, 3]
>>> x
[1, 2, 3]

Un diccionario es como una lista, pero sus elementos no están ordenados y el “índice” que me permite acceder a cada uno puede ser cualquier tipo, no necesariamente un entero. Típicamente es un string. Por ejemplo:

>>> datos = {}
>>> datos["nombre"] = "Manolo"
>>> datos["telefono"] = "999999999"
>>> datos["nombre"]
'Manolo'

El diccionario también es mutable (se puede asignar otro valor a cualquiera de sus elementos), y cuando se asigna una variable de tipo diccionario a otra, en realidad se copian las referencias, como en las listas, por lo que ambas variables se refieren al mismo diccionario y modificar un dato a través de una de ellas, afecta a la otra (como en las listas):

>>> x = datos
>>> x["nombre"]="Juan"
>>> datos["nombre"]
'Juan'

Si queremos obtener una copia de un diccionario para operar sobre la copia sin afectar al original, haríamos x = dict(datos).

Las cadenas y las tuplas, aunque comparten con las listas la sintaxis para acceder a sus elementos individuales, son sin embargo INMUTABLES. Si intentamos cambiar uno de sus elementos tendremos un error.

>>> x = (1,2,3)
>>> x[0]
1
>>> x[0] = 9
Traceback (most recent call last):
  File "<interactive input>", line 1, in <module>
TypeError: 'tuple' object does not support item assignment

3.1.5 Métodos de cadenas y listas

La mayoría de los tipos están implementados en realidad como un objeto y tienen métodos que permiten hacer operaciones sobre él. El intérprete puede proporcionar ayuda interactiva sobre los métodos soportados por un tipo concreto, poniendo help(tipo). Por ejemplo, probar a poner help(str) para ver métodos de las cadenas, o help(list) para los de las listas. Si en vez de todos ellos solo queremos ver uno, podemos poner help(tipo.metodo). Una vez vistos, probar algunos de ellos:

Cadenas:

>>> t = "hola"
>>> t.upper()
'HOLA'
>>> t.capitalize()
'Hola'
>>> t.isdigit()
False
>>> t.startswith("ho")
True

Listas:

>>> l = [1,2,3]
>>> l.reverse()
>>> l
[3, 2, 1]
>>> l.sort()
>>> l
[1, 2, 3]
>>> l.count(2)
1
>>> l.append(5)
>>> l
[1, 2, 3, 5]

Conversiones entre listas y cadenas (ver help(str.join) y help(str.split)):

>>> l = list(t)
>>> l
['h', 'o', 'l', 'a']
>>> " + ".join(l)
'h + o + l + a'
>>> "".join(l)
'hola'
>> frase = "Esta frase tiene varias palabras"
>> l = frase.split()
>> l
['Esta', 'frase', 'tiene', 'varias', 'palabras']
>> ";".join(l)
'Esta;frase;tiene;varias;palabras'
>> l = frase.split(";")
['Esta', 'frase', 'tiene', 'varias', 'palabras']

3.2 Control de flujo

En python el “bloque” de sentencias no se delimita mediante ningún carácter especial (a diferencia de la mayoría de los lenguajes que sí lo hacen, por ejemplo Java o C usan llaves). En vez de eso, usa la indentación del bloque como elemento sintáctico para delimitar el bloque (a diferencia de los restantes lenguajes en los que la indentación es ignorada por el compilador).

Por esta razón, estos ejemplos se prueban mejor en el editor que en el intérprete interactivo (aunque también se podría), ya que el editor automáticamente indenta el código cuando detecta que comenzamos un nuevo bloque.

3.3 Condicionales

Hacer los ejemplos siguientes para comprender la sintaxis y funcionamiento:

a = 2
b = 3
if a>b:
    print("la variable a es mayor")
    print("ya que a=%d y b=%d" % (a,b))
print("Fin de programa")

Cambiar a por 1 y volver a ejecutar.

a = 2
b = 3
if a>b:
    print("la variable a es mayor")
else:
    print "la variable b es mayor"
print("ya que a=%d y b=%d" % (a,b))
print("Fin de programa")

Otro ejemplo:

a = 2
b = 2
if a > b:
    print("la variable a es mayor")
elif b > a:
    print("la variable b es mayor")
else:
    print("a y b son iguales")
print("ya que a=%d y b=%d" % (a,b))
print("Fin de programa")

Python no tiene switch(), debe implementarse mediante elifs.

3.3.1 Bucles

Python solo tiene el bucle while (no tiene el do..while ni el repeat..until).

i = 1
while i<=3:
    print(i)
    i = i + 1

Que puede usarse para recorrer listas, o cadenas, o tuplas:

t = "Hola"
i = 0
while i < len(t):
    print(t[i])
    i = i + 1

Pero además tiene una potente construcción llamada el iterador (for). El iterador permite que una variable vaya tomando valores sucesivos extraídos de una lista, o cadena o tupla y es el equivalente al foreach de otros lenguajes de programación:

texto = "Hola"
for letra in texto:
    print(letra.upper())

Otro ejemplo:

numeros = [1,6,4,9,20,3]
mayor = numeros[0]
for numero in numeros:
    if numero>mayor:
        mayor=numero
print(mayor)

A menudo for se usa en combinación con la función range() (que devuelve una lista de valores consecutivos, consulta help(range) para más opciones) para obtener una serie de números seguidos. Usado así puede servir como sustituto del bucle for del C o Java.

for n in range(30):
    if n%5==0:
        print(n)

3.3.2 Funciones

Las funciones se definen con def, y se debe especificar la lista de parámetros que van a recibir (aunque no sus tipos). El cuerpo de la función irá delimitado por indentación. La función puede contener return en cualquier lugar, y se puede retornar cualquier tipo de dato (retornar una tupla sirve para devolver varios resultados).

El paso de parámetros no es más que una asignación del parámetro actual al parámetro formal. Por tanto el paso es por copia y una función no puede modificar el dato que recibe.

Sin embargo, si el parámetro es una lista, ya vimos que sí que puede modificar alguno de sus elementos, y esa modificación afectaría al programa principal, aunque en general no es una práctica recomendable.

def maximo(numeros):
    max = numeros[0]
    for n in numeros:
        if n > max:
            max = n
    return max

datos = [2,3,15,23,2,1]
m = maximo(datos)
print(m)

La siguiente función recibe una lista de datos y devuelve otra lista copia de la original en la que se han eliminado los ceros.

def quitarceros(datos):  # No modifica la lista original
    nueva = []
    for d in datos:
        if d != 0:
            nueva.append(d)
    return nueva

datos = [1,5,0,3,0,12,0,0]
datos2 = quitarceros(datos)
print(datos2)
print(datos)

3.4 Clases y objetos

Una clase define un prototipo que tiene campos con datos (como las estructuras C), y tiene funciones (métodos). La idea es la misma que las clases C++ o Java. También tiene un constructor que puede inicializar los campos del objeto (llamado __init__). Dentro de los métodos de la clase para referirse al objeto concreto que la invoca, se usa el parámetro llamado self (el equivalente al this del C++ o el Java), que siempre hay que declarar como primer parámetro del método. ¡Acordarse de usar self.campo, y no simplemente campo como en Java! Al invocar los métodos del objeto, el parámetro self no se pasa, sino que el intérprete ya lo inicializa con la referencia al objeto que lo está invocando, es decir, en x.metodo(), self sería x.

Ejemplo (observar que el constructor en este ejemplo recibe parámetros con valores por defecto, lo que significa que pueden omitirse en la llamada).

class Complejo:
    real = 0
    imag = 0
    def __init__(self, re=0, im=0):
        self.real = re
        self.imag = im
    def imprimir(self):
        print("%.2f + %.2fi" % (self.real, self.imag))

Una vez declarada la clase, el nombre de la clase se convierte en una función que al llamarla invoca al constructor y retorna una referencia al objeto recién creado (mientras se ejecuta el constructor, self apunta a ese objeto recién creado también). Así Complejo(1,1) crea un objeto del tipo Complejo e inicializa sus campos con los valores 1 y 1.

y = Complejo(1,1)
y.imprimir()    # Mostrará 1.0 + 1.0i

3.4.1 Herencia

Una clase puede heredar de otra, y así recibir sus atributos y métodos, que después podrá extender o redefinir (sobrecargar). Como muestra el simple ejemplo siguiente, la sintaxis de la herencia consiste en escribir la clase base entre paréntesis:

class Imaginario(Complejo):   # La clase de la que hereda va entre paréntesis
    def __init__(self, im=0):
        self.real = 0
        self.imag = im

z = Imaginario(3)
z.imprimir()  # Mostrará 0.00 + 3.00i

Python admite herencia múltiple, con las complejidades que ésto puede acarrear, especialmente al tratar de invocar el constructor de la clase base. Estos problemas se salen del ámbito de esta breve introducción y sólo se verán en su momento si son necesarios en alguna práctica.

3.5 Módulos

Un módulo no es más que un fichero que contiene código python, normalmente para implementar clases y funciones. La idea es que ese código pueda “incluirse” en otro programa para poder así acceder a las clases y funciones allí definidas, cumpliendo por tanto el papel de las bibliotecas en otros lenguajes.

La sintaxis más sencilla para usar un módulo es mediante import. Por ejemplo, el módulo math contiene funciones matemáticas, y mediante import math tendremos acceso a esas funciones. Puedes averiguar qué funciones y clases implementa un módulo haciendo help(math), por ejemplo, desde el intérprete, pero sólo funcionará si has hecho antes import math (ya que sin eso el símbolo math no está definido).

Una vez que el módulo ha sido importado, esto crea un espacio de nombres con el nombre del módulo, dentro del cual están todas las clases, funciones y variables implementadas en el módulo. Esto implica que para acceder a esas clases, funciones y variables hay que escribir delante el nombre de módulo y un punto. Ejemplo:

import math
print(math.sqrt(5))
print(math.pi)

Hay sintaxis alternativas para simplificar el uso de estos espacios de nombres. Por ejemplo, si el módulo importado tiene un nombre demasiado largo de escribir, podemos “renombrarlo” cuando lo importamos así:

import math as m
print(m.sqrt(5))
printr(m.pi)

Si sólo vamos a necesitar una función del módulo, podemos importar esa función solo. En este caso, para usarla no requeriremos precederla del espacio de nombres:

from math import sqrt
print(sqrt(5))

Finalmente, se puede usar el asterisco para importar TODAS las funciones, clases y variables definidas en un módulo, y así no tener que usar el espacio de nombres para usarlas. Este “truco” se ve a menudo en ejemplos de código de Internet, pero se desaconseja porque los espacios de nombres ayudan a evitar colisiones en nombres de funciones y variables.

3.5.1 Escribir un módulo que pueda ejecutarse como programa

Cuando python carga un módulo, ejecuta todo su contenido. Normalmente el módulo contiene sólo definiciones de funciones y clases, pero podría contener asignaciones de variables o cualquier otro código ejecutable, el cual se ejecutaría cuando el módulo es cargado. Por ejemplo:

Escribe el siguiente código en un fichero y guárdalo con el nombre mimodulo.py.

# Guarda este fichero con el nombre mimodulo.py
def saludo(nombre):
    print("Hola", nombre)

print("Modulo cargado")
x = 3

Si en la misma carpeta tienes otro programa que haga import mimodulo, al ejecutar este último5 veremos aparecer en la consola el texto “Modulo cargado” y además quedarán definidas la función mimodulo.saludo y la variable mimodulo.x con el valor 3.

Además, python también inicializa una variable automáticamente llamada mimodulo.__name__ con el valor mimodulo. Sin embargo, si desde línea de comandos hacemos python mimodulo, también se ejecutará el módulo y veremos aparecer el mensaje, pero en este caso el valor que python asigna a mimodulo.__name__ no es "mimodulo", sino la cadena especial "__main__".

Esto nos sirve para distinguir desde dentro del módulo si estamos siendo ejecutados como consecuencia de un import, o directamente como programa principal. Así podemos cambiar nuestro comportamiento según sea el caso:

def saludo(nombre):
    print("Hola", nombre)

if __name__ == "__main__":
    print("Ejecutado directamente como programa principal")
else:
    print("Modulo cargado")
    x = 3

Prueba a ejecutarlo como programa principal (si estás en IEP pulsa Ctrl+E, si no ve a la interfaz de comandos y teclea python mimodulo.py). Deberás ver cómo sale el mensaje que indica que ha sido ejecutado como programa principal.

Prueba en cambio a ejecutar el intérprete solo (o cambia al panel de intérprete si estás en IEP) y en él escribe import mimodulo. O bien escribe otro pequeño programa principal que contenga import mimodulo. En este caso deberá aparecer el otro mensaje.

Esto permite escribir módulos que definan funciones útiles para ser usadas en otros programas, y que a la vez contengan una especie de “demo” de su utilidad que pueda ser ejecutada directamente como programa principal. Por ejemplo:

def saludo(nombre):
    print("Hola", nombre)

if __name__ == "__main__":
    nombre = raw_input("Como te llamas? ")
    saludo(nombre)

3.6 Documentación

Hemos dejado para el final uno de los aspectos más importantes de la programación en cualquier lenguaje, y es la generación de documentación del programa que se esté escribiendo. Python admite dos formas de documentación:

3.6.1 Comentarios

Como cualquier otro lenguaje, python permite escribir comentarios que serán ignorados por el intérprete. En el caso de python, todo lo que siga al carácter # será ignorado.

3.6.2 Docstrings

Además de los comentarios que son ignorados, cada módulo y cada función dentro del módulo puede llevar asociada una cadena de documentación, (docstring) que no son ignoradas por el intérprete, sino que son utilizadas para generar la ayuda que podemos ver con help() y eventualmente puede ser transformada en HTML para generar una documentación imprimible, al estilo de Javadoc.

Si queremos usar estas cadenas de documentación, basta añadirlas como la primera sentencia de una función, o de un módulo. Ya que habitualmente estas cadenas ocuparán varias líneas, se suele usar el delimitador """ (triple) para contenerlas. Este delimitador no es exclusivo de los docstring aunque es donde más suele verse, sino que es parte de la sintaxis de python y puede usarse en cualquier lugar en que se necesite delimitar una cadena que ocupe varias líneas. Observa el ejemplo siguiente:

# Este es el modulo saludo.py
"""Este modulo implementa funciones relacionadas con saludos."""
def saludo(nombre):
    """Emite por la salida estandar un saludo al nombre que
    recibe como parametro"""
    print("Hola", nombre)

La documentación incorporada de este modo está accesible a través de help() cuando se carga el módulo.

>>> import saludo
>>> help(saludo)
Help on module saludo:

NAME
    saludo - Este modulo implementa funciones relacionadas con saludos.

FILE
    /carpeta/donde/esta/guardado/saludo.py

FUNCTIONS
    saludo(nombre)
        Emite por la salida estandar un saludo al nombre que
        recibe como parametro

>>>

3.7 Decoradores

Un decorador es una función que toma como parámetro otra función, y devuelve como resultado una tercera función. Aplicar un decorador a una función permite alterar cómo se comporta esta, sin necesidad de tocar su código.

Como el concepto es complejo, comencemos haciendo un ejemplo que muestra cómo lograr alterar una función sin necesitar decoradoes. Esto es posible porque en Python las funciones también son objetos por lo que se pueden recibir como parámetro y devolver como resultado.

Supongamos que tengo la función siguiente:

def cuadrado(n):
    return n*n

La forma de invocarla sería por ejemplo r = cuadrado(5). Ahora imagina que queremos que la función, antes de ejecutarse, imprima “Voy a ejecutar cuadrado” y cuando haya terminado, que imprima “He ejecutado cuadrado”. No queremos cambiar la función original, de modo que escribimos otra que la llame:

def cuadrado_modificada(n):
    print("Voy a ejecutar cuadrado")
    result  = cuadrado(n)
    print("He ejecutado cuadrado")
    return result

Esto está mejor, pero no es muy general. Si tuviéramos diferentes funciones (cuadrado, raiz, etc.) y quisiéramos “instrumentar” (añadir código como acabamos de hacer) cualquiera de ellas, tendríamos que programar muchas funciones como cuadrado_modificada(), raiz_modificada(), etc. todas ellas prácticamente iguales. En cuanto nos encontremos copiando y pegando trozos del código en otra parte para modificarlos, debería sonar una alarma de que puede hacerse mejor.

Queremos hacer una función más genérica, que reciba como parámetro la función a instrumentar y la llame, en un sandwich entre los print(). Podemos averiguar además el nombre de la función que estamos invocando para mostrarlo en el print() usando funcion.__name__ (¿dijimos que las funciones son objetos?). Sería así:

def funcion_modificada(funcion, dato):
    print("Voy a invocar", funcion.__name__)
    result = funcion(dato)
    print("He invocado", funcion.__name__)
    return result

Esto lo invocaríamos con r = funcion_modificada(cuadrado, 5), por ejemplo.

Esto está mejor, pero aún no es lo bastante general, porque ¿y si la función a instrumentar tiene más de un parámetro? La función anterior no nos valdría y habría que hacer otra, muy similar, pero con más parámetros.

Python soporta una sintaxis especial a la hora de listar los parámetros de una función. Si el parámetro comienza por *, esto indica que se trata en realidad de una lista de parámetros, con cualquier número de ellos. Es decir, una función así:

def muchos_parametros(*args):
    ...

Cuando es invocada así:

muchos_parametros(1,2,3,4)

Recibe en realidad en su único parámetro args una lista [1,2,3,4]. De forma simétrica, si tenemos una lista como l = [1,2,3,4] y queremos pasar los datos individuales a una función, en lugar de la lista, podemos hacer funcion(*l). Entonces la función recibirá cuatro parámetros (obviamente debe estar declarada para aceptar cuatro paránetros, o tener un truco tipo *args). Comprueba la diferencia entre print(l), que imprime la lista, o print(*l), que imprime los valores.

Con este truco podemos mejorar nuestra función instrumentadora6:

def funcion_modificada(funcion, *args):
    print("Voy a invocar", funcion.__name__)
    result = funcion(*args)
    print("He invocado", funcion.__name__)
    return resultado

Ya casi lo tenemos. Pero ahora quiero que mi antigua función cuadrado() sea en realidad la versión modificada. Es decir, no quiero tener que llamar así: r = funcion_modificada(cuadrado, 5), sino que quiero llamar así: r = cuadrado(5), tal y como la llamaba antes, pero que se ejecute en su lugar la versión instrumentada.

¿Cómo lograr esto? Se puede gracias a que en Python las funciones no son más que variables que apuntan al objeto función que van a ejecutar. Es decir, yo puedo hacer:

otro = cuadrado
print(otro(7))  # Y sale 49

Por tanto la idea parece que va por la línea de:

viejo_cuadrado = cuadrado
cuadrado = funcion_modificada

pero aún no está bien. Para empezar, hemos perdido la función cuadrado original (por eso he guardado una referencia a la original en la variable viejo_cuadrado) pero además no hemos ganado nada, pues la nueva cuadrado necesita invocarse como funcion_modificada, es decir, así: r = cuadrado(viejo_cuadrado, 5).

No obstante vamos por el buen camino. Lo que quiero es que funcion_modificada, en lugar de ejecutar la función que se le pasa, cree a partir de ella una nueva función y retorne esa nueva función como resultado. Así (la cosa se complica un poco):

def decorar_con_prints(funcion):
    def version_decorada(*args):
        print("Voy a invocar", funcion.__name__)
        result = funcion(*args)
        print("He invocado", funcion.__name__)
    return version_decorada

Ahora puedo hacer desde el programa principal:

cuadrado = decorar_con_prints(cuadrado)
print(cuadrado(5))

Y se mostrará:

Voy a invocar cuadrado
He invocado cuadrado
25

Lo que acabamos de implementar es lo que se llama un decorador (tomado del Patrón Decorador). Python introduce el azúcar sintáctico para que, una vez que he escrito un decorador (la función decorar_con_prints), sea muy fácil aplicárselo a otras funciones que escriba luego. Basta poner @nombre_del_decorador delante de la función que vaya a implementar.

Por ejemplo, si quiero implementar una función suma() que también sea ejecutada por el decorador, escribiría:

@decorar_con_prints
def suma(a, b):
    return a+b

Y puedo invocarla así: r = suma(2,3). Prueba incluso r = cuadrado(suma(2,3)).

Es decir, la sintaxis:

@decorador
def funcion(...):
    ...

es interpretada por python como equivalente a:

funcion = decorador(funcion)

Observa que este patrón es muy poderoso, pues la función que es retornada por el decorador no tiene por qué invocar internamente a la función que recibe como parámetro, sino que puede retornar cualquier otra función. El decorador a su vez puede recibir parámetros que no tiene por qué pasar a la función decorada.

Cuando un decorador lleva parámetros, como aquí:

@meta_decorador(parametro)
def funcion(...):
    ...

Python lo ejecuta como si hubiéramos puesto:

decorador = meta_decorador(parametro)
funcion = decorador(funcion)

Esto ya es material avanzado que se sale de los objetivos de la asignatura, pero en este caso se entiende que la función decorador retornaría otra función que a su vez sería un decorador (quizás uno diferente según el parámetro que se le haya pasado).

Los decoradores son utilizados intensivamente por los frameworks de desarrollo web, como una forma de asociar funciones con rutas HTTP. Veremos ejemplos de su uso en futuras sesiones de prácticas. Es posible usar los decoradores sin entender exactamente cómo funcionan, pero los ejemplos anteriores habrán servido para hacerse una idea.

El siguiente ejemplo de código puede ayudarte a entender de qué forma los decoradores son usados en este tipo de frameworks. Quizás no lo entiendas ahora, pero se aconseja revisitarlo tras la práctica sobre Web.

# Variable global donde iremos guardando las funciones registradas
funciones = {}

def registrar(categoria):
    # La categoría sería una cadena para catalogar las funciones
    def decorador(funcion):
        # El decorador almacena en un diccionario el nombre de la función
        if categoria not in funciones:
            funciones[categoria] = []
        funciones[categoria].append(funcion.__name__)
        # Pero simplmemente retorna la misma función sin cambios
        return funcion
    # Retornar el decorador
    return decorador


@registrar("aritmetica")
def suma(a,b):
    return a+b

@registrar("aritmetica")
def resta(a,b):
    return a-b

@registrar("aritmetica")
def producto(a,b):
    return a*b

@registrar("logica")
def son_iguales(a,b):
    return a==b

@registrar("logica")
def son_distintas(a,b):
    return a!=b

# Los decoradores, al ejecutarse, van guardando información sobre las
# funciones registradas, aún si éstas no han sido ejecutadas aún (porque
# el decorador sí ha sido ejecutado)

print(funciones)
# Resultado:
# {'aritmetica': ['suma', 'resta', 'producto'], 
#  'logica': ['son_iguales', 'son_distintas']}

Mucho del material de ejemplo sobre python que puedes encontrar en Internet es para la rama 2.

Hay muchas diferencias a nivel interno, en el manejo de las clases, etc. Una de las diferencias más importantes que nos atañe de cara a la programación de aplicaciones de red es el manejo de las cadenas. Para manejar secuencias de bytes sin significado concreto python3 tiene el nuevo tipo bytes. Ya que la API de sockets (y la de ficheros) lo único que transporta son bytes, será necesario convertir de Unicode a bytes o viceversa para transmitir cadenas por la red. Esta conversión es más explícita y menos proclive a errores en Python3. Iremos viendo estas pequeñas diferencias a medida que las necesitemos.↩︎
