3.4 Clases y objetos

Una clase define un prototipo que tiene campos con datos (como las estructuras C), y tiene funciones (m√©todos). La idea es la misma que las clases C++ o Java. Tambi√©n tiene un constructor que puede inicializar los campos del objeto (llamado __init__). Dentro de los m√©todos de la clase para referirse al objeto concreto que la invoca, se usa el par√°metro llamado self (el equivalente al this del C++ o el Java), que siempre hay que declarar como primer par√°metro del m√©todo. ¬°Acordarse de usar self.campo, y no simplemente campo como en Java! Al invocar los m√©todos del objeto, el par√°metro self no se pasa, sino que el int√©rprete ya lo inicializa con la referencia al objeto que lo est√° invocando, es decir, en x.metodo(), self ser√≠a x.

Ejemplo (observar que el constructor en este ejemplo recibe par√°metros con valores por defecto, lo que significa que pueden omitirse en la llamada).

    imag = 0
    def __init__(self, re=0, im=0):
        self.real = re
        self.imag = im
    def imprimir(self):
        print("%.2f + %.2fi" % (self.real, self.imag))

Una vez declarada la clase, el nombre de la clase se convierte en una funci√≥n que al llamarla invoca al constructor y retorna una referencia al objeto reci√©n creado (mientras se ejecuta el constructor, self apunta a ese objeto reci√©n creado tambi√©n). As√≠ Complejo(1,1) crea un objeto del tipo Complejo e inicializa sus campos con los valores 1 y 1.

3.5 M√≥dulos

Un m√≥dulo no es m√°s que un fichero que contiene c√≥digo python, normalmente para implementar clases y funciones. La idea es que ese c√≥digo pueda ‚Äúincluirse‚Äù en otro programa para poder as√≠ acceder a las clases y funciones all√≠ definidas, cumpliendo por tanto el papel de las bibliotecas en otros lenguajes.

La sintaxis m√°s sencilla para usar un m√≥dulo es mediante import. Por ejemplo, el m√≥dulo math contiene funciones matem√°ticas, y mediante import math tendremos acceso a esas funciones. Puedes averiguar qu√© funciones y clases implementa un m√≥dulo haciendo help(math), por ejemplo, desde el int√©rprete, pero s√≥lo funcionar√° si has hecho antes import math (ya que sin eso el s√≠mbolo math no est√° definido).

Una vez que el m√≥dulo ha sido importado, esto crea un espacio de nombres con el nombre del m√≥dulo, dentro del cual est√°n todas las clases, funciones y variables implementadas en el m√≥dulo. Hay sintaxis alternativas para simplificar el uso de estos espacios de nombres. Por ejemplo, si el m√≥dulo importado tiene un nombre demasiado largo de escribir, podemos ‚Äúrenombrarlo‚Äù cuando lo importamos as√≠:

Si s√≥lo vamos a necesitar una funci√≥n del m√≥dulo, podemos importar esa funci√≥n solo. En este caso, para usarla no requeriremos precederla del espacio de nombres:

Finalmente, se puede usar el asterisco para importar TODAS las funciones, clases y variables definidas en un m√≥dulo, y as√≠ no tener que usar el espacio de nombres para usarlas. Este ‚Äútruco‚Äù se ve a menudo en ejemplos de c√≥digo de Internet, pero se desaconseja porque los espacios de nombres ayudan a evitar colisiones en nombres de funciones y variables.

3.7 Decoradores

Un decorador es una funci√≥n que toma como par√°metro otra funci√≥n, y devuelve como resultado una tercera funci√≥n. Aplicar un decorador a una funci√≥n permite alterar c√≥mo se comporta esta, sin necesidad de tocar su c√≥digo.

Como el concepto es complejo, comencemos haciendo un ejemplo que muestra c√≥mo lograr alterar una funci√≥n sin necesitar decoradoes. Esto es posible porque en Python las funciones tambi√©n son objetos por lo que se pueden recibir como par√°metro y devolver como resultado.

Supongamos que tengo la funci√≥n siguiente:

def cuadrado(n):
    return n*n

La forma de invocarla ser√≠a por ejemplo r = cuadrado(5). Ahora imagina que queremos que la funci√≥n, antes de ejecutarse, imprima ‚ÄúVoy a ejecutar cuadrado‚Äù y cuando haya terminado, que imprima ‚ÄúHe ejecutado cuadrado‚Äù. No queremos cambiar la funci√≥n original, de modo que escribimos otra que la llame:

def cuadrado_modificada(n):
    print("Voy a ejecutar cuadrado")
    result  = cuadrado(n)
    print("He ejecutado cuadrado")
    return result

Esto est√° mejor, pero no es muy general. Si tuvi√©ramos diferentes funciones (cuadrado, raiz, etc.) y quisi√©ramos ‚Äúinstrumentar‚Äù (a√±adir c√≥digo como acabamos de hacer) cualquiera de ellas, tendr√≠amos que programar muchas funciones como cuadrado_modificada(), raiz_modificada(), etc. todas ellas pr√°cticamente iguales. En cuanto nos encontremos copiando y pegando trozos del c√≥digo en otra parte para modificarlos, deber√≠a sonar una alarma de que puede hacerse mejor.

Queremos hacer una funci√≥n m√°s gen√©rica, que reciba como par√°metro la funci√≥n a instrumentar y la llame, en un sandwich entre los print(). Podemos averiguar adem√°s el nombre de la funci√≥n que estamos invocando para mostrarlo en el print() usando funcion.__name__ (¬ødijimos que las funciones son objetos?). Ser√≠a as√≠:

def funcion_modificada(funcion, dato):
    print("Voy a invocar", funcion.__name__)
    result = funcion(dato)
    print("He invocado", funcion.__name__)
    return result

Esto lo invocar√≠amos con r = funcion_modificada(cuadrado, 5), por ejemplo.

Esto est√° mejor, pero a√∫n no es lo bastante general, porque ¬øy si la funci√≥n a instrumentar tiene m√°s de un par√°metro? La funci√≥n anterior no nos valdr√≠a y habr√≠a que hacer otra, muy similar, pero con m√°s par√°metros.

Python soporta una sintaxis especial a la hora de listar los par√°metros de una funci√≥n. Si el par√°metro comienza por *, esto indica que se trata en realidad de una lista de par√°metros, con cualquier n√∫mero de ellos. Es decir, una funci√≥n as√≠:

def muchos_parametros(*args):
    ...

Cuando es invocada as√≠:

muchos_parametros(1,2,3,4)

Recibe en realidad en su √∫nico par√°metro args una lista [1,2,3,4]. De forma sim√©trica, si tenemos una lista como l = [1,2,3,4] y queremos pasar los datos individuales a una funci√≥n, en lugar de la lista, podemos hacer funcion(*l). Entonces la funci√≥n recibir√° cuatro par√°metros (obviamente debe estar declarada para aceptar cuatro par√°netros, o tener un truco tipo *args). Comprueba la diferencia entre print(l), que imprime la lista, o print(*l), que imprime los valores.

Con este truco podemos mejorar nuestra funci√≥n instrumentadora6:

def funcion_modificada(funcion, *args):
    print("Voy a invocar", funcion.__name__)
    result = funcion(*args)
    print("He invocado", funcion.__name__)
    return resultado

Ya casi lo tenemos. Pero ahora quiero que mi antigua funci√≥n cuadrado() sea en realidad la versi√≥n modificada. Es decir, no quiero tener que llamar as√≠: r = funcion_modificada(cuadrado, 5), sino que quiero llamar as√≠: r = cuadrado(5), tal y como la llamaba antes, pero que se ejecute en su lugar la versi√≥n instrumentada.

¬øC√≥mo lograr esto? Se puede gracias a que en Python las funciones no son m√°s que variables que apuntan al objeto funci√≥n que van a ejecutar. Es decir, yo puedo hacer:

print(otro(7))  # Y sale 49

Por tanto la idea parece que va por la l√≠nea de:

viejo_cuadrado = cuadrado
cuadrado = funcion_modificada

pero a√∫n no est√° bien. Para empezar, hemos perdido la funci√≥n cuadrado original (por eso he guardado una referencia a la original en la variable viejo_cuadrado) pero adem√°s no hemos ganado nada, pues la nueva cuadrado necesita invocarse como funcion_modificada, es decir, as√≠: r = cuadrado(viejo_cuadrado, 5).

No obstante vamos por el buen camino. Lo que quiero es que funcion_modificada, en lugar de ejecutar la funci√≥n que se le pasa, cree a partir de ella una nueva funci√≥n y retorne esa nueva funci√≥n como resultado. As√≠ (la cosa se complica un poco):

def decorar_con_prints(funcion):
    def version_decorada(*args):
        print("Voy a invocar", funcion.__name__)
        result = funcion(*args)
        print("He invocado", funcion.__name__)
    return version_decorada

Ahora puedo hacer desde el programa principal:

print(cuadrado(5))

Y se mostrar√°:

Voy a invocar cuadrado
He invocado cuadrado
25

Lo que acabamos de implementar es lo que se llama un decorador (tomado del Patr√≥n Decorador). Python introduce el az√∫car sint√°ctico para que, una vez que he escrito un decorador (la funci√≥n decorar_con_prints), sea muy f√°cil aplic√°rselo a otras funciones que escriba luego. Basta poner @nombre_del_decorador delante de la funci√≥n que vaya a implementar.

Por ejemplo, si quiero implementar una funci√≥n suma() que tambi√©n sea ejecutada por el decorador, escribir√≠a:

@decorar_con_prints
def suma(a, b):
    return a+b

Y puedo invocarla as√≠: r = suma(2,3). Prueba incluso r = cuadrado(suma(2,3)).

Es decir, la sintaxis:

@decorador
def funcion(...):
    ...

es interpretada por python como equivalente a:

funcion = decorador(funcion)

Observa que este patr√≥n es muy poderoso, pues la funci√≥n que es retornada por el decorador no tiene por qu√© invocar internamente a la funci√≥n que recibe como par√°metro, sino que puede retornar cualquier otra funci√≥n. El decorador a su vez puede recibir par√°metros que no tiene por qu√© pasar a la funci√≥n decorada.

Cuando un decorador lleva par√°metros, como aqu√≠:

@meta_decorador(parametro)
def funcion(...):
    ...

Python lo ejecuta como si hubi√©ramos puesto:

decorador = meta_decorador(parametro)
funcion = decorador(funcion)

Esto ya es material avanzado que se sale de los objetivos de la asignatura, pero en este caso se entiende que la funci√≥n decorador retornar√≠a otra funci√≥n que a su vez ser√≠a un decorador (quiz√°s uno diferente seg√∫n el par√°metro que se le haya pasado).

Los decoradores son utilizados intensivamente por los frameworks de desarrollo web, como una forma de asociar funciones con rutas HTTP. Veremos ejemplos de su uso en futuras sesiones de pr√°cticas. Es posible usar los decoradores sin entender exactamente c√≥mo funcionan, pero los ejemplos anteriores habr√°n servido para hacerse una idea.

El siguiente ejemplo de c√≥digo puede ayudarte a entender de qu√© forma los decoradores son usados en este tipo de frameworks. Quiz√°s no lo entiendas ahora, pero se aconseja revisitarlo tras la pr√°ctica sobre Web.

# Variable global donde iremos guardando las funciones registradas
funciones = {}

def registrar(categoria):
    # La categor√≠a ser√≠a una cadena para catalogar las funciones
    def decorador(funcion):
        # El decorador almacena en un diccionario el nombre de la funci√≥n
        if categoria not in funciones:
            funciones[categoria] = []
        funciones[categoria].append(funcion.__name__)
        # Pero simplmemente retorna la misma funci√≥n sin cambios
        return funcion
    # Retornar el decorador
    return decorador


@registrar("aritmetica")
def suma(a,b):
    return a+b

@registrar("aritmetica")
def resta(a,b):
    return a-b

@registrar("aritmetica")
def producto(a,b):
    return a*b

@registrar("logica")
def son_iguales(a,b):
    return a==b

@registrar("logica")
def son_distintas(a,b):
    return a!=b

# Los decoradores, al ejecutarse, van guardando informaci√≥n sobre las
# funciones registradas, a√∫n si √©stas no han sido ejecutadas a√∫n (porque
# el decorador s√≠ ha sido ejecutado)

print(funciones)
# Resultado:
# {'aritmetica': ['suma', 'resta', 'producto'], 
#  'logica': ['son_iguales', 'son_distintas']}
